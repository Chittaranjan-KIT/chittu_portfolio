System Design

Three-layer architecture

A natural-language brief drives a generate → evaluate → optimize loop across two coupled physics domains. The cross-domain coupling — aero shape affecting cooling airflow affecting battery temperature — is the systems-engineering differentiator.

Layer 3 · Interface

Natural-language design brief → target specs

Parses intent ("low drag, long range, generous cooling") into engineering constraints and explains the tradeoffs. Optional Anthropic-API upgrade for free-form reasoning.

▼
Layer 1 · Generative engine

Conditional VAE → parametric geometry

Learns the feasible design manifold and generates new 3D-parametric concepts (body + thermal/packaging params) conditioned on target performance.

▼
Layer 2 · Aero surrogate

Gradient Boosting → Cd

Predicts drag instantly, replacing CFD in the loop.

Layer 2 · Thermal surrogate

Gradient Boosting → peak battery temp

Predicts cooling sufficiency, replacing Simulink in the loop.

Methodology & Honesty

Scope chosen like an engineering manager

A deep, validated narrow project beats a shallow broad one. The fidelity choices are deliberate — and stating them precisely is itself an engineering-maturity signal.

Surrogates are bootstrapped on validated empirical/analytical correlations (a fast CFD/Simulink stand-in — exactly how surrogates are meant to be trained). Top designs are then verified against an independent physics evaluation to quantify surrogate error.

Production validation path: OpenFOAM / STAR-CCM+ for aerodynamics and a Simulink lumped-capacitance model for thermal. The method is identical; only the fidelity of the reference changes.
